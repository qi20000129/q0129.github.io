第三章 面向对象三大特征—封装

1.封装的概念

封装是指将对象的状态信息隐藏在对象内部，不允许外部程序直接访问对象内部信息，而是通过该类所提供的方法来实现内部信息的操做访问。

2.封装的步骤

(1).把属性私有化（PRIVATE）
(2).对这些属性设置GET和SET方法
(3).调用处理这些方法

3.封装的作用

避免一些用一些不合乎常理的操作。

4.修饰符

PUBLIC—>公共的：在一个项目中都可以访问，权限最高
PRIVATE—>私有的：只能在本类中进行访问，权限最低
PROTECTED—>安全的，受保护的：同一个包能进行访问，继承中之类可以访问父类
DEFAULT—>默认的：同一个包能访问

1.集合框架的概念

集合框架是为表示和操作而规定的一种统一的标准体系结构。

2.List接口，Set接口，Map接口的区别

SET接口继承COLLECTION接口，存储一组唯一(不允许重复)，无序的对象。
LIST接口继承COLLECTION接口，存储一组不唯一(允许重复)，有序的对象。
MAP接口存储一组成对的键-值对象，提供KEY(键)到VALUE(键)的映射。MAP中的KEY不要求有序，不允许重复。VALUE同样不要求有序，但允许重复。

第九章 高级功能—–文件操作

1.文件概述

文件是具有内容和用途的，而文件夹是用来放置这些文件的。

2.什么是流

JAVA流，是指一串流动的字符，是以先进先出的方式发送和接收数据的通道。

3.写文件

(1)字节输出流写文件
INT WRITE(INT C)—–写入一个字节数据
INT WRITE(BYTE[ ] BUF)—–写入数组BUF的所有字节
INT WRITE(BYTE[ ] B,INT OFF,INT LEN)—–将字节数组从OFF位置开始，长度为LEN的字节数据输出大宋输出流中
VOID CLOSE( )—–关闭输出流
(2)字符输出流写文件
WRITE(STRING STR)—–将STR字符串里包含的字符输出到指定的输出流中
WRITE(STRING STR,INT OFF,INT LEN)—–将STR字符串里从OFF位置开始长度为LEN的字符输出到输出流中
VOID CLOSE()—–关闭输出流
VOID FLUSH()—–刷新输出流

4.读文件

(1)字节输入流读文件
INT READ()—–从输入流中读取单个字符
INT READ(BYTE[ ] B)—–将数据读取到字节数组中
INT READ(BYTE[ ] B,INT OFF,INT LEN)—–从输入流中读取最多LEN长度的字符，保存到字符数组B中，保存的位置从OFF位置开始
VOID CLOSE( )—–关闭输入流
INT AVAILABLE()—–返回输入流读取的估计字节数
(2)字符输入流读文件
INT READ()—–从输入流中读取单个字符
INT READ(BYTE[ ] B)—–从输入流中读取C.LENGTH长度的字符，保存到字符数组C中，返回实际读取的字符数
READ(BYTE[ ] C,INT OFF,INT LEN)—–从输入流中读取最多LEN长度的字符，保存到字符数组C中，保存的位置从OFF位置开始，返回实际读取的字符数
VOID CLOSE()—–关闭流

第三章，操作表中数据

1，SQL命令中的表达式和逻辑运算符

1.1表达式

在数据库中的表达式一般由常量，变量，函数和运算符组成。注意：单个常量，变量或函数亦可称做表达式。
SQL语言中包括三种表达反式：
第一种：<表名> 后跟<字段名表达式>
第二种：select语句后的<目标表达式>
第三种：where语句后的<条件表达式>

1.2比较运算符

比较运算符用来比较两个数的大小，其结果是1，0或者null。
运算符 含义
> 大于
>= 大于等于
< 小于
<= 小于等于
= 等于
!=或<> 不等于
is null 是否为空
is not null 是否不为空
in 判断一个值是in列表中的任意一个值
not in 判断一个值不是in列表中的任意一个值
like 通配符匹配
between and 判断一个值是否在两个值之间

1.3逻辑运算符

SQL命令支持的运算符有And(或者&&)，Or(或者||)，Not(或者!)。 逻辑运算符用来判断表达式的真假。如果表达式是真，结果返回 1。如果表达式是假，结果返回 0。
And 和Or运算符连接条件，Not否定条件。And连接条件，并且仅有当了两个条件都为真时才返回1，Or也连接两个条件，但只要任何一个为真就返回1。
当一个语句中使用了多个逻辑运算符时，运算先求Not的值，然后求And的值，最后求Or的值。

MYSQL第二章总结

2019-10-22

第二章数据库和表的设计

1，为什么需要数据完整性？

数据的完整性是指数据的精准性和可靠性。数据完整性类型一般分四种：

1.实体完整性：通过唯一约束，主键约束或标识列属性来实现，如：PRIMARY KEY（主键约束）、UNIQUE 唯一值约束。

2.域完整性：通过数据类型（如数值类型、日期类型、字符串类型等限制类型）、非空约束(NOT NULL)、默认值定义（DEFAULT）、CHECK检查(MySQL不支持check约束)来实现。

3.引用完整性：又叫做参照完整性，通过外键约束（FOREIGN KEY）来实现。引用完整性保证键值在所有表中一致，不能引用不存在的值。

4.自定义完整性：是指针对某一具体关系型数据库的约束条件，它反映某一具体应用所涉及的数据必须满足的语义要求。

2，数据完整性在MySQL中的体现

数据约束：主键，主键自增，外键，非空，默认，唯一。

1，主键：primary key 特性：非空且唯一，写在字段类型后面。

在MySQL中设置字段做主键时，分两种情况。
（1）定义列的同时设置主键
（2）完成所有列之后设置主键

2，主键自增：auto_increment 写在字段类型后面，主键前面。

在设置auto_increment 时需要遵守以下几点规则：
（1）一个表中只能有一个字段使用auto_increment。
（2）auto_increment 必须作为主键的一个部分，不得单独使用。
（3） auto_increment 字段的数据类必须为整数类型。

3，外键：从表的外键，必须在主表的主键中出现，写在创建语句的最后面。

外键约束中有两个名词，需要注意：
主表：对于两个具有关系的表而言，相关联的字段中主键所在的表即为主表。
从表：对于两个具有关系的表而言，相关联的字段中外键所在的表即为从表。

MySQL第1章总结

2019-10-20

第一章MySQL基础

1，MySQL的特点？

①运行速度：mysql的运行速度相当快，是目前最快的数据库系统之一。
②易用性：MySQL是一个相对简单的，高性能的数据库系统，与其他大型数据库系统相 比，它的设置和管理要简单的多。
③SQL支持：它支持结构化查询语言，是目前所有现代数据库系统都选用的标准语言。
④功能性：MySQL服务器支持多线程，因此允许多个客户端同时与它链接。每个客户端可 以使用多个数据库。
⑤连接性和安全性：MySQL完全支持网络化，用户可以从因特网任何地点访问数据库，它 还拥有访问控制功能，可以保证链接的一方完全无法看到另一方的数据。
⑥可移植性：MySQL除了可以在Windows上运行外，还可以运行在Unix和Linux系统上， 能在小型设备上运行也可以在高端服务器上运行。
⑦可用性和成本：MySQL是一个开源项目，有许多条款可供选择。
⑧公开发行和源代码：获取MySQL的方法很简单，只要使用浏览器即可。
